A few honest caveats.

Context for how to read everything below. None of it changes the engineering; it changes what you should and should not expect from a 3-day unpaid build.

  1. Built on the newest possible stack, deliberately optimized for cost. In production the image generation here would be proprietary models behind paid API calls. For an unpaid showcase I instead used Higgsfield's free generations plus a small tool I wrote to automate them. The engineering decisions are identical; only the billing model differs.
  2. The tools change every few months. The reasoning does not. This build exists to show I can deliver regardless of which models are SOTA this quarter. Identity preservation today is SAM3 segmentation + a hi-res reference inpaint; not long ago the same goal meant SDXL/Flux + a LoRA, or IP-Adapter / InstantID / PuLID / FaceID inpaint on the face, a skin detailer to remove plastic skin, then a separate upscaler. Same problem, same logic, different tools. That transfer is the point.
  3. This is a proof-of-work, not a finished product. It is not perfect and I know exactly where it is rough and how I would harden each step for real production. Taking it there properly takes real time and effort, which I would rather not spend unpaid on a showcase for a role that is still undecided. If you want further proof that I can bring this to genuine production quality, I am glad to, on a paid basis.
  4. Images and videos are shown compressed, not at native resolution. They are downscaled and re-encoded to keep the page light on free static hosting. Originals were generated at 2K resolution.
The pipeline

Reference in, recognizable character out.

Multi-model, multi-stage, with a deliberate identity-preservation step in the middle. The hard part is keeping the same person after every edit pass, and although it never will be pixel perfect, it can be good enough.

01

Reference generation

Nano Banana Pro (2K, 9:16) on Higgsfield, identity locked to a canonical hero + ground-truth face/body set.

02

Photoreal edit

Qwen Rapid AIO v23, text-driven editing tuned for photorealism: skin texture, framing, scene.

03

Head segmentation

SAM3 in ComfyUI segments the head, 1:1 square crop, geometry written to JSON.

04

Hi-res face inpaint

Crop re-rendered at higher resolution against the original face reference.

05

Feathered paste-back

Edited face downscaled into the recorded region (shrinking kills blur), seam feathered. Identity restored.

06

Image upscaling

SeedVR2 image upscale back to 2K resolution for the final deliverable.

Identity preservation, the part that matters

Primitive face restoration flow.

Primitive, but it works! I treat identity as a reproducible post-process: segment, crop, restore at high res against the locked reference, then paste back by recorded geometry. stdlib-only Python drives ComfyUI over its HTTP API.

# settings.py - one place for every option, CLI flags override
SAM_PROMPT     = "female head"   # SAM3 segmentation target
FORCE_SQUARE   = True           # 1:1 crop, best for upscalers
FEATHER        = 24             # seam blend width (px)
FACE_SUFFIX    = "_face"         # filename is the join key between stages

# stage 1: photo -> output/<key>/<key>_face.png + _coords.json
# (edit / skin / SeedVR2 upscale happen on the crop here)
# stage 2: edited face -> downscale into recorded region -> feather -> final/
$ python3 batch_face_crop.py   # 32 images, sequential, unattended
$ python3 batch_paste_back.py  # key = filename minus _face
Restored crops: segmented input → edited face

Why downscale the edited face instead of upscale the target: shrinking a high-res render into the slot adds detail and preserves identity (skin, eye color, etc.). That is the reason behind it. Screenshots below.

Tooling & the cost call

To semi-automate generations locally and save money...

I quickly vibe-coded Higgie, a browser extension for Higgsfield that queues tens of prompts with their relevant reference images and runs them unattended, so this whole project cost nothing in image generation.

Higgie: queue saturator

A browser extension I vibe-coded to keep a Higgsfield unlimited-mode queue full automatically: load a prompt list + reference images, set a count, walk away. Hundreds of prepared prompts generated without a human clicking.

The cost decision

Bulk generation ran on an owned Higgsfield unlimited plan instead of a metered image API. Editing, skin detail, SAM3 segmentation, inpaint and SeedVR2 upscale ran on a local ComfyUI GPU. The expensive, repeated step is the one I moved off per-call billing. Same call you make choosing local GPUs vs an aggregator by volume.

Prompt architecture

Not "good prompts." A fixed appearance lock + a reusable realism/imperfection stack appended to every generation + a hero-reference-first method, organized into a per-content-pillar prompt library.
